# 1

**adaptability** ->
Uma estrutura de alto nível permite a personalização para atender às necessidades de:

-   pequenas startups
-   grandes organizações complexas, como empresas de software e holdings

**scope flexibilty**  ->

-   as organizações definem o escopo do SGSI (Sistema de Gestão de Segurança da Informação) para alinhá-lo às suas operações e às expectativas das partes interessadas

**customer focus**  ->

-   o escopo reflete o que os clientes e as partes interessadas precisam com base em seus interesses


<a id="org32093b1"></a>

# 2

**technology & cloud service providers** ->

-   Gerenciam grandes volumes de dados de clientes
-   A certificação distingue empresas preocupadas com a segurança em mercados competitivos
-   Tranquiliza os órgãos reguladores quanto à proteção proativa de dados

**financial services** ->

-   Protege dados financeiros altamente sensíveis (informações bancárias e de seguros)

**healthcare**  ->

-   Concentra-se na proteção de dados de saúde de pacientes, que são um alvo principal de ataques


<a id="org8843cf6"></a>

# 3

**regulatory landscape** ->

-   Certos setores (serviços financeiros) têm obrigações legais em relação à segurança da informação.

**client and partner commitments** ->

-   A certificação costuma ser um requisito básico para a aquisição de clientes.
-   Reduz o ônus das auditorias de segurança e dos questionários de due diligence.

**geographical footprint** ->

-   Para operações globais, incluir escritórios internacionais no escopo do SGSI pode aumentar a garantia quanto à abrangência dos controles implementados.

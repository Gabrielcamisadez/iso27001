# purpose of application (i.e., scoping questionnarie)

-   Processo formal para iniciar a auditoria de certificação ISO e solicitar orçamento a um organismo de certificação
-   Coleta informações sobre o escopo, a complexidade e o contexto organizacional do SGSI (Sistema de Gestão de Segurança da Informação)


<a id="org4add4bf"></a>

# outcome

-   Auxilia o organismo de certificação no planejamento de auditorias com os recursos adequados e dentro de suas áreas técnicas


<a id="orgb5b3af8"></a>

# step 1 - scoping call

-   Reunião inicial para discutir o escopo geral, preços e cronograma


<a id="org0d29ae7"></a>

# step 2 - formal application submission

-   O solicitante preenche um formulário detalhado com todas as informações relevantes


<a id="org177bec0"></a>

# step 3 - application review

-   O organismo de certificação verifica se as informações estão completas e corrige inconsistências
-   Confirma se o escopo está dentro da área técnica
-   Apresenta os preços em uma proposta


<a id="org0cbf548"></a>

# organizational context

-   size
-   industry
-   locations


<a id="org74e7d25"></a>

# scope

-   Limites da certificação, incluindo escritórios, departamentos e processos


<a id="org615e309"></a>

# audit criteria

-   Determina quais normas estão sendo buscadas (por exemplo, ISO 27001, ISO 27701, ISO 42001)


<a id="org26799ac"></a>

# previous 3 party assurance

-   Identifica sobreposições ou prontidão com base em avaliações anteriores (por exemplo, SOC 2)


<a id="orgafea8c4"></a>

# timeline

O candidato deve comunicar quaisquer restrições de cronograma ou prazos.


<a id="org896803e"></a>

# prior compliance initiatives

-   Aproveita as sobreposições com auditorias anteriores (por exemplo, SOC 2) para otimizar os processos


<a id="org1cc231f"></a>

# statement of applicability

-   Verifica os controles justificados e identifica as áreas que requerem testes


<a id="org16719b1"></a>

# audit methodology

-   Determina se a auditoria remota ou presencial é apropriada com base na estrutura organizacional


<a id="orgc704fdf"></a>

# scope of the audit

-   Normas, número de funcionários, locais, departamentos, processos e produtos
-   Garante o tamanho e a competência adequados da equipe de auditoria


<a id="org583b082"></a>

# risk and complexity

-   Avalia riscos externos, como a sensibilidade dos dados e os requisitos regulatórios


<a id="org9c84bcc"></a>

# audit time

-   Estima o tempo total de auditoria necessário para avaliar o escopo com base nas informações da solicitação e nos requisitos de acreditação


<a id="orgcbfe388"></a>

## take the application submission seriously


<a id="org120c019"></a>

# clarity in scope

-   Evita confusão e ineficiências tanto para os auditores quanto para as organizações solicitantes


<a id="org6a5b06f"></a>

# adequate audit team

-   Garante que o tamanho e a competência da equipe correspondam ao escopo, evitando o desperdício de recursos


<a id="org5fa7780"></a>

# impact of poor scope definition

-   Escopos amplos ou indefinidos podem levar a falhas de comunicação, retrabalho e auditorias ineficientes

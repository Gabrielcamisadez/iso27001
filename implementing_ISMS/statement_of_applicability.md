# SoA

Ocorre após a análise inicial de riscos
A SOA, exigida pela cláusula 6.1.3, detalha:

-   controles incluídos ou excluídos do SGSI
-   justificativas para inclusão ou exclusão e status de implementação de cada controle


<a id="orgff8fdd3"></a>

## Records Control Mapping to risks

-   os controles mitigam os riscos identificados durante a avaliação de riscos e devem ser documentados na SOA


<a id="org69f7a04"></a>

## guide implementation

-   identifica o status de implementação do controle, garantindo que os recursos sejam direcionados de forma eficaz


<a id="org5fdcd44"></a>

## audit evidence

-   essencial para que os auditores verifiquem se os controles necessários estão em vigor ou planejados


<a id="orgdf0f20b"></a>

## supports business continuity

-   serve como um registro transparente e responsável dos controles do SGSIs


<a id="orgcbd365a"></a>

# Anatomy of the SOA

-   nome e número do controle (ex.: A.6.7 - Trabalho Remoto)
-   aplicável (Dentro do Escopo) ou Não Aplicável (Fora do Escopo)
-   **justificativa** &#x2013;> explicação para inclusão ou exclusão do escopo
-   Status de implementação &#x2013;> implementado, parcialmente implementado ou não implementado


<a id="org375a666"></a>

# common pitfalls

**omissão de controles necessários** ->

-   a ausência de controles do Anexo A pode levar a problemas de auditoria Resultados e incidentes de segurança

**Justificativas pouco claras/inadequadas** ->

-   Garantir que as justificativas de exclusão sejam claras e compreensíveis para as partes interessadas

**Revisão ou atualizações para melhoria** ->

-   Revisões regulares mantêm a SOA relevante e eficaz
